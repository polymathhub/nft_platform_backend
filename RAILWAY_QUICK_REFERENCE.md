# Production FastAPI Deployment - Quick Reference

## Problem Solved

**Error**: `Could not import module "main"`

**Root Cause**: 
- Using `main:app` instead of `app.main:app`
- Wrong working directory in container
- Missing module imports validation

**Solution**: All provided files use correct path `app.main:app` and validate imports

## Quick Deploy (5 minutes)

### Step 1: Install Railway CLI
```bash
npm install -g @railway/cli
```

### Step 2: Login
```bash
railway login
```

### Step 3: Initialize
```bash
cd nft_platform_backend
railway init
```

### Step 4: Add Services
```bash
railway add
# Select: PostgreSQL
```

### Step 5: Configure
```bash
railway variable add SECRET_KEY "your-secret-key"
railway variable add ENVIRONMENT "production"
railway variable add LOG_LEVEL "info"
```

### Step 6: Deploy
```bash
railway up
```

### Step 7: Test
```bash
railway logs -f
curl https://your-railway-app.railway.app/health
```

**Done!** Your FastAPI backend is deployed to Railway. ✅

## Local Testing (Before Deployment)

### Option 1: Docker Run
```bash
# Build locally
docker build -f Dockerfile.production -t nft-platform:latest .

# Run locally with test database
docker run -p 8000:8000 \
  -e DATABASE_URL="postgresql://user:pass@localhost:5432/nft_test" \
  nft-platform:latest

# Test
curl http://localhost:8000/health
```

### Option 2: Docker Compose
```bash
# Start all services (PostgreSQL + Redis + App)
docker-compose -f docker-compose.production.yml up -d

# View logs
docker-compose -f docker-compose.production.yml logs -f app

# Test
curl http://localhost:8000/health

# Stop
docker-compose -f docker-compose.production.yml down
```

## Critical Fixes Applied

### ✅ Fix 1: Correct Module Path
```dockerfile
# WRONG ✗
CMD ["uvicorn", "main:app", "--host", "0.0.0.0"]

# CORRECT ✓ (provided)
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0"]
```

### ✅ Fix 2: Import Validation
```dockerfile
# Validates before container starts
RUN python -c "from app.main import app; print('✓ Imported successfully')" || exit 1
```

### ✅ Fix 3: Working Directory
```dockerfile
WORKDIR /app
# Now all imports work: from app.main import app
```

### ✅ Fix 4: Database Migrations
```dockerfile
CMD ["sh", "-c", "alembic upgrade head && uvicorn app.main:app ..."]
```

### ✅ Fix 5: Health Checks
```dockerfile
HEALTHCHECK --interval=30s --timeout=10s --start-period=15s --retries=3 \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health').read()"
```

### ✅ Fix 6: Environment Variables
```dockerfile
ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    WORKERS=4
```

## Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| `Could not import "main"` | Wrong module path | Use `app.main:app` ✓ (provided) |
| `Health check failing` | `/health` endpoint down | Already exists in your code |
| `Database connection error` | `DATABASE_URL` not set | Run `railway add` for PostgreSQL |
| `Migrations not running` | Alembic not executed | Dockerfile runs it automatically |
| `Module not found` | Dependencies not installed | Check `requirements.txt` |

Complete troubleshooting guide in `RAILWAY_PRODUCTION_GUIDE.md`.
